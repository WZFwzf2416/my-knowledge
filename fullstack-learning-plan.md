# 前端转全栈学习路线规划

## 目标

用一个真实可上线的项目，完整走一遍：

- 产品设计
- 前端开发
- 后端开发
- 数据库设计
- 用户登录鉴权
- 文件上传
- 部署上线
- 域名绑定
- 基础运维与迭代

这份规划的核心目标不是“把后端理论一次学完”，而是先具备独立做出并上线一个全栈产品的能力。

---

## 推荐路线

建议按下面的顺序学习，而不是一开始就做重型前后端分离项目：

1. 先做一体化全栈项目
2. 完成真实部署上线
3. 补齐工程化、监控、环境变量、生产问题处理
4. 再做前后端分离项目
5. 最后再补 Docker、CI/CD、云服务器运维

原因：

- 前端开发者上手更快
- 学习反馈更快，容易形成正循环
- 可以先理解服务端思维，再上复杂架构
- 能尽快拿到一个真实作品

---

## 第一阶段最推荐的技术栈

### 主路线

- 前端/全栈框架：Next.js
- UI：Tailwind CSS
- 数据库：PostgreSQL
- ORM：Prisma
- 鉴权：Supabase Auth
- 文件存储：Supabase Storage
- 部署平台：Vercel
- 数据库托管：Supabase
- 表单校验：Zod

### 为什么推荐这套

- Next.js 对前端开发者非常友好，能同时学页面、接口和服务端渲染
- PostgreSQL 是非常值得学的主流关系型数据库
- Prisma 学习曲线平滑，适合第一套全栈项目
- Supabase 一站式提供数据库、鉴权、存储，能减少基础设施负担
- Vercel 部署体验好，适合快速上线真实项目

---

## 不建议一开始就选的路线

下面这套适合第二个项目，不建议作为第一套：

- React + Vite
- NestJS
- MySQL / PostgreSQL
- Redis
- Docker
- Nginx
- Linux 云服务器

原因不是这套不好，而是初期同时学习这些内容，容易被环境、部署和架构复杂度拖慢。

## 推荐练手项目

最推荐你做下面这个项目：

## 项目题目

个人知识库 / 收藏夹系统

### 核心功能

- 用户注册登录
- 新建知识卡片
- 编辑和删除知识卡片
- 标签分类
- 搜索
- 收藏/置顶
- 图片上传
- 公开/私密状态
- 个人资料页

### 为什么适合练手

- 典型 CRUD 很完整
- 能覆盖用户系统和权限控制
- 能练数据库设计
- 能练文件上传和搜索
- 上线后自己也能长期使用

---

## 推荐学习顺序

## 第 1 阶段：做出第一个可上线版本

目标：2 周内完成 MVP 并上线

### 要学的内容

- Next.js App Router 基础
- 路由、布局、页面组织
- 服务端与客户端组件的区别
- 表单提交
- Route Handlers 或 Server Actions
- PostgreSQL 基础
- Prisma 基础
- Supabase 项目初始化
- 登录注册流程
- Vercel 部署

### 交付结果

- 能注册登录
- 能创建、编辑、删除内容
- 能在数据库中保存数据
- 有一个线上可访问地址

---

## 第 2 阶段：补齐“像产品”的能力

目标：第 3 到第 4 周

### 要补的能力

- 表单校验
- 权限控制
- 错误处理
- Loading / Empty / Error 状态
- 搜索与筛选
- 分页
- 详情页优化
- 基础 SEO

### 交付结果

- 项目不只是“能跑”，而是具备基本产品质量
- 用户体验明显更完整
- 代码结构更清晰

---

## 第 3 阶段：补部署与工程化

目标：第 5 周

### 要学的内容

- 环境变量区分本地与生产
- Migration 管理
- 域名绑定
- HTTPS
- 日志与错误排查
- Preview / Production 环境差异
- 基础性能优化

### 交付结果

- 你知道如何把项目稳定部署到线上
- 你知道线上出问题时应该从哪里排查

---

## 第 4 阶段：做第二个项目，进入前后端分离

目标：第 6 周及以后

### 推荐技术栈

- 前端：React + Vite 或 Next.js 纯前端层
- 后端：NestJS
- 数据库：PostgreSQL
- ORM：Prisma
- 缓存：Redis
- 部署：前端 Vercel，后端 Railway / Render

### 这个阶段重点学习

- REST API 设计
- 分层架构
- JWT / Session
- CORS
- DTO / 参数校验
- 后端模块化设计
- 独立部署前后端服务

---

## 你应该重点掌握的知识地图

### 前端层

## 学习节奏建议

### 每周固定安排

- 3 天开发功能
- 1 天补后端和数据库理论
- 1 天处理部署、排错、复盘
- 周末做总结并记录踩坑

### 每次开发都要记录

- 今天做了什么
- 遇到了什么 bug
- 为什么会出这个问题
- 最后怎么解决
- 下次如何避免

这样成长会非常快，因为你不只是写代码，而是在积累自己的工程经验。

---

## 学习时的优先级

### 优先级最高

- 先上线
- 先打通核心链路
- 先理解请求到数据库再到页面的完整流程

### 不要过早纠结

- 复杂架构设计
- 过度抽象
- 高级状态管理
- 微服务
- 过早上 Docker 和 K8s

## 6 周执行版本

### 第 1 周

- 创建 Next.js 项目
- 配置 Supabase
- 配置 Prisma
- 设计数据库表
- 完成登录注册

### 第 2 周

- 完成知识卡片 CRUD
- 完成标签系统
- 部署到 Vercel

### 第 3 周

- 增加搜索、筛选、分页
- 加入 Zod 表单校验
- 完善错误处理

### 第 4 周

- 加图片上传
- 加公开/私密
- 加个人资料页

### 第 5 周

- 自定义域名
- 调整 SEO
- 完善环境变量和 migration
- 做一次性能与体验优化

### 第 6 周

- 复盘整个项目
- 重新梳理数据库设计
- 开始第二个前后端分离项目

---

## 最终目标

完成这条路线后，你应该具备以下能力：

- 能独立做一个全栈项目
- 能把项目部署到线上
- 能处理基本数据库设计
- 能实现登录与权限控制
- 能完成真实环境下的联调与排错
- 能开始学习更正式的后端框架，如 NestJS

---

## 下一步建议

建议立刻开始下面这件事：

1. 确定项目题目为“个人知识库 / 收藏夹系统”
2. 用 Next.js 初始化项目
3. 连接 Supabase 和 Prisma
4. 先做登录和第一张数据表
5. 一周内完成最小可用版本

不要先追求“大而全”，先让项目真实跑起来。
